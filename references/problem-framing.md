# Problem Framing

Use this first when the request is vague, overloaded, or still mixing diagnosis with solutioning.

## Checklist

- State the deliverable in one line.
- State the exact question in one line.
- State what decision this research is supposed to unblock.
- Name the evidence sources and why each one matters.
- State what has already been tried.
- State what must not change or what assumptions are fixed.

## Prompt Skeleton

```text
Deliverable:
<diagnosis memo | research brief | literature synthesis | anomaly analysis | theory check>

Question:
<what exactly needs to be determined>

Decision to unblock:
<what this analysis will help decide>

Materials map:
- <path>: <role>
- <path>: <role>

What has already been tried:
- <attempt>
- <attempt>

Constraints:
- <scope, budget, platform, or evidence limits>

Required output:
- Findings
- Supporting evidence
- Remaining uncertainty
- Recommended next steps
```
